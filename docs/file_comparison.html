File Comparison with FlashFinder

Back to Main Menu

FlashFinder's file comparison feature allows users to easily identify differences between two files. Whether you're comparing text documents, images, or other file types, this guide will help you understand how the comparison feature works.

Basic File Comparison

To start a basic comparison:

  1. Select the two PDF files you want to compare.
  2. Right-click and choose the "Compare" option.
  3. FlashFinder will highlight the differences between the files using the specified output options.

Handling Large Files

Comparing large files might take a bit longer. Larger page sizes also have an effect on processing times and may take longer.

High DPI levels will also result in long processing times. Choose a level that suits your needs best.

Navigating the Compare Settings

FlashFinder has numerous options to customize the file comparison output

Output

There are several output options the control how the comparison file is output.

  • DPI Level stands for "Dots Per Inch" aka "Pixels Per Inch". It is essentially the resolution of the output. Higher DPI settings mean more pixels to process resulting in longer processing times but a higher quality output

    • Note: It is reccommended to use the lowest DPI setting you require to ensure timely comparison times.

    WARNING: High DPI settings use high amounts of RAM, combined with large files (or large pages), This can be a problem in some cases.

  • Page size sets the output comparison files page size. This can be important when printing the comparison.

    • Note: The output page size has minimal effect on the processing time.

  • Output path determines where the comparison file will be saved once complete.

    • Source path: The output file is saved to the folder location where the original files were saved.

    Default path: The output file is saved to the "Documents" folder.

    Specified path: The output file is saved to the path specified in the "Specified Path" bar.

Files to Include

The comaprison tool has a few page options that will show different details to aid in analyses of the files.

  • New copy: The output file will contain the newest version of the original files.
  • Old copy: The output file will contain the older version of the original files.
  • Markup: The output file will contain a marked up version of the newest file.

    • Markups include green boxes indicating large changes (Focus points to analyse further) and red highlights to indicate all other changes between the files.

  • Difference: The output file will contain a minimalist difference copy showing only changes between the files.

    • This is essentially taking the newest file and subtracting the older file leaving only the pixels that have changed left behind.

    The differences are shown with blue highlights being the newer file (Added changes over the older file) and red highlights indicating the older file (Things that have been removed in the newer file).

  • Overlay: The output file will contain a the older file in red with the newer file overlayed on top.

    • The newer file is layed on top of a red version of the old file which results in any removed items shown in red.

Note: "New" and "Old" are in reference with the default settings where the "Main Page" setting (See Main Page) is set to "New Document".

If the setting "Main Page" is set to "Old Document" then the Markup, Difference, and Overlay pages will use the older file as the main file and the opposite will be true.

Ex. "Markup" will markup the older file rather then the newer file.

General

The comparison tool has several options to further customize the output.

  • Scale pages: Scales the original document pages to be the set page size prior to comparing.

    • This setting helps when comparing files that do not share the same page size. This however can reduce the quality of the comparison if scaling a large page size to a small page size. (Reduces file size)

    This setting can also be used if comparison results are not as expected.

    FlashFinder will try to auto correct any comparison where page sizes are not the same. Results may vary in this case.

  • Black and white: Sets the output pages to be in black and white. (Reduces file size)
  • Grayscale: Similar to black and white but allows for gray hues that can occur when mixing black and white (shading). (Reduces file size)
  • Reduce size: Reduces the output file size which a small loss in quality. Useful when trying to share the comparison file.

Main Page

The main page setting controls which file is the focus when comparing. As noted in the Files to Include section, the comparison outputs differently based on this setting

  • New document (Default): Sets the focus to be the newest of the files.
  • Old document: Sets the focus to be the oldest of the files.
