Advanced Searching with FlashFinder

Back to Main Menu

FlashFinder provides a powerful mechanism for file search, catering to generic string searches, regex searches, and command searches. This guide will delve into the specifics of these search techniques and the settings you can customize.

General

FlashFinder supports searching all directories or you can search the current directory simply by toggling between the modes using the button to the right of the search bar.

Generic String Searches

For basic searches, simply enter the desired keyword or phrase. FlashFinder will search for files and directories containing the entered text.

Regex Searches

If you're familiar with regular expressions, FlashFinder allows for more advanced pattern-based searches. To initiate a regex search, start your search term with double colons [::]. For example, ::\d{4} will search for any sequence of four digits in file names.

See the Regex page for more information on regular expressions

Command Searches

Command searches in FlashFinder provide a way to refine your search criteria using specific commands:

  • contains and !contain: Check if a filename contains or does not contain specific terms.
  • equals and !equal: Exact matches or exclusions based on specific terms.
  • type and !type: Filter or exclude files based on their types or extensions.

The suffixes for the commands :all and :any: alter the way the commands use the terms.

Examples:

  • Find files containing "apple" AND "recipe": contains(apple, recipe), contains:all(apple, recipe)
  • Find files containing "apple" OR "recipe": contains:any(apple, recipe)

Command searches can also be used in combination with other commands. You can also use as many terms in the commands as youd like.

Examples:

  • Find files that contain "resume" and are a PDF: contains(resume), type(pdf)
  • Find files that contains "business", "finance", "engineering", "art", or "science": equals:any(business, finance, engineering, art, science)

Search Settings

FlashFinder offers customization of the search process through its settings (found in the "search" tab in the settings window):

Include Subfolders

By default, FlashFinder includes subfolders in search results. If you wish to exclude subfolders, uncheck the "Include subfolders in search results" option in the settings.

Indexed Search

To accelerate the search process, FlashFinder can use indexed data. This can make searches faster, but the results might not be the most up-to-date. Toggle the "Use indexes when searching" option in the settings to enable or disable this feature.

Cached Search

To further speed up the search process, FlashFinder will cache the index data into memory resulting in significantly faster search times but much greater memory usage (~750MB/1,000,000 indexed files).
